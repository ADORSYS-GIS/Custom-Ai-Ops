# How-to guides

**Task-oriented.** A how-to guide is a recipe for a competent reader who knows what they want: "Enable KV offloading with LMCache", "GPU-enable a cluster with DRA", "Autoscale vLLM with KEDA".

Rules ([Diátaxis: how-to guides](https://diataxis.fr/how-to-guides/)):

- Starts with prerequisites and the end state.
- Numbered steps, copy-pasteable commands, expected output shown.
- No teaching, no theory — link to `../explanation/` for the why.
- State what versions/environment the guide was validated against, and when.

Naming: verb-first slug, e.g. `enable-lmcache-cpu-offload.md`, `deploy-vllm-on-k8s.md`.
