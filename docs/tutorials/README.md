# Tutorials

**Learning-oriented.** A tutorial takes a newcomer by the hand through a complete, guaranteed-success experience. The reader learns by doing; the goal is confidence and orientation, not completeness.

Rules ([Diátaxis: tutorials](https://diataxis.fr/tutorials/)):

- Must work first try, from a stated starting point (OS, hardware, versions pinned).
- No choices, no detours, no "alternatively you could…". One happy path.
- Explain almost nothing — link to `../explanation/` instead.
- Tested end-to-end by someone other than the author before merge.

Naming: `NN-short-slug.md` in learning order, e.g. `01-first-vllm-serve.md`, `02-vllm-on-k8s.md`.
