# Explanation

**Understanding-oriented.** Explanations give context, background, and the *why*: "Why naive serving fails", "The KV cache economy", "Why round-robin load balancing breaks for LLMs" ([Diátaxis: explanation](https://diataxis.fr/explanation/)).

Rules:

- No step-by-step instructions (that's `../how-to/`), no exhaustive tables (that's `../reference/`).
- Every performance claim links to a report in `../benchmarks/` or an external source.
- Write for the engineer who will join the team next year.

Note: the existing `../explain/` directory predates this structure; its content is being consolidated here — see [`../MIGRATION.md`](../MIGRATION.md).
